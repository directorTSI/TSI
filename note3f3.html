Introduction: Variables & Operators

Operators are the backbone of computation and logic in Python. They are special symbols that perform operations on values and variables, allowing you to manipulate data, make comparisons, and control program flow. This lesson will demystify the various categories of operators and revisit the crucial concept of variables, providing clear definitions and practical examples.

1. Variables Explained

Variables are fundamental to programming, acting as named containers that store and manage data within your Python scripts, making your code dynamic and understandable.

Definition: Named storage locations in memory.

Purpose: Store diverse information (numbers, text, collections), refer to data by meaningful names, and enhance code flexibility.

Analogy: A labeled box holding items - the label is the variable name, the item is its value.

2. Crafting Valid Python Variables: Naming Rules

Understanding the rules for naming variables is crucial for writing error-free and readable Python code.

Rule 1: Start with a Letter or Underscore

Variable names MUST begin with an alphabet character (a-z, A-Z) or an underscore (_).

# Valid Examples
my_variable = 10
_temp_value = 25.5
name = "Alice"

print("my_variable: {}".format(my_variable))
print("_temp_value: {}".format(_temp_value))
print("name: {}".format(name))
                            
my_variable: 10
_temp_value: 25.5
name: Alice
# Invalid Example: Starts with a number
# 1st_number = 5
# print(1st_number)
                            
# SyntaxError: invalid decimal literal

Rule 2: Cannot Start with a Number

A variable name CANNOT begin with a digit (0-9).

# Valid Examples
count1 = 100
data_2024 = "Report"

print("count1: {}".format(count1))
print("data_2024: {}".format(data_2024))
                            
count1: 100
data_2024: Report
# Invalid Example: Starts with a number
# 7days = "week"
# print(7days)
                            
# SyntaxError: invalid decimal literal

Rule 3: Alpha-Numeric & Underscores Only

Variable names can ONLY consist of letters (a-z, A-Z), numbers (0-9), and underscores (_). No spaces or other special characters allowed.

# Valid Examples
total_score = 95
user_id_123 = "john_doe"

print("total_score: {}".format(total_score))
print("user_id_123: {}".format(user_id_123))
                            
total_score: 95
user_id_123: john_doe
# Invalid Examples: Contains hyphen, space, or special character
# my-variable = 50
# first name = "Bob"
# price$ = 20.00
                            
# SyntaxError: can't assign to operator
# SyntaxError: invalid syntax
# SyntaxError: invalid syntax

Rule 4: Case-Sensitive

Python distinguishes between uppercase and lowercase letters. 'myVariable' is different from 'myvariable'.

score = 100
Score = 50

print("score: {}".format(score))
print("Score: {}".format(Score))
                            
score: 100
Score: 50

Rule 5: Cannot Be Keywords

Python's reserved keywords (e.g., if, for, def, True) have special meanings and cannot be used as variable names.

# Invalid Example
# if_keyword = 10 # 'if' is a keyword

# Valid Example
my_if_condition = True
print("my_if_condition: {}".format(my_if_condition))
                            
my_if_condition: True

3. Best Practices for Variable Names

Adhering to best practices makes your code more readable, maintainable, and less prone to errors.

  • Be Descriptive: Names should clearly indicate purpose (e.g., student_name, total_price).
  • Use snake_case: For multi-word names, use lowercase with underscores (e.g., first_name, items_in_cart). This is the Python standard (PEP 8).
  • Avoid Ambiguity: Choose names that are easy to understand and distinguish from each other.
  • Consistency is Key: Stick to a uniform naming style throughout your project.

4. Arithmetic Operators: The Math Wizards

These operators are used to perform common mathematical operations such as addition, subtraction, multiplication, and division. They are the fundamental tools for numerical computations in Python.

Addition (+) & Unary Plus

Used to add two operands or indicate a positive number.

# Adding values
print("Addition: {}".format(2 + 5))

# Unary Plus (indicates positive, usually implicit)
print("Unary Plus: {}".format(+1))
                            
Addition: 7
Unary Plus: 1

Subtraction (-) & Unary Minus

Used to subtract the right operand from the left, or indicate a negative number.

# Subtracting values
print("Subtraction: {}".format(6 - 4))

# Unary Minus (indicates negative)
print("Unary Minus: {}".format(-1))
                            
Subtraction: 2
Unary Minus: -1

Multiplication (*)

Used to multiply two operands.

result = 7 * 3
print("Multiplication: {}".format(result))
                            
Multiplication: 21

Division (/)

Performs division and always returns a float result, even if the division is exact.

X = 10
Y = 20
print("X/Y: {}".format(X / Y))
print("Y/X: {}".format(Y / X)) # Note: still a float!
                            
X/Y: 0.5
Y/X: 2.0

Floor Division (//)

Performs division but returns the integer part of the quotient (rounds down to the nearest whole number).

x_val = 18
y_val = 20
print("Y // X: {}".format(y_val // x_val)) # 20 divided by 18 is 1.11..., floor is 1
print("7 // 2: {}".format(7 // 2))       # 7 divided by 2 is 3.5, floor is 3
                            
Y // X: 1
7 // 2: 3

Remainder Modulus (%)

Performs division but returns the remainder of the division.

print("3 % 2: {}".format(3 % 2)) # 3 divided by 2 is 1 with remainder 1
print("10 % 3: {}".format(10 % 3)) # 10 divided by 3 is 3 with remainder 1
print("15 % 4: {}".format(15 % 4)) # 15 divided by 4 is 3 with remainder 3
                            
1
1
3

Power (**)

Raises the first operand to the power of the second operand (exponentiation).

print("2 ** 3: {}".format(2 ** 3)) # 2 to the power of 3 (2*2*2)
print("5 ** 2: {}".format(5 ** 2)) # 5 to the power of 2 (5*5)
                            
8
25

5. Operator Precedence & Associativity

When multiple operators are in an expression, Python follows specific rules to determine the order of evaluation. Precedence dictates which operator is evaluated first, while Associativity defines the order when operators have the same precedence.

Precedence Hierarchy (Highest to Lowest)

Operators higher in the list are evaluated before those lower down.

Operators Associativity Description
() Left to Right Parentheses (Grouping)
** Right to Left Exponentiation
+x, -x Left to Right Unary Plus, Unary Minus
*, /, //, % Left to Right Multiplication, Division, Floor Division, Modulus
+, - Left to Right Addition, Subtraction
==, !=, >, <, >=, <= Left to Right Comparison Operators
not Right to Left Logical NOT
and Left to Right Logical AND
or Left to Right Logical OR
=, +=, -=, etc. Right to Left Assignment Operators

Step-by-Step Evaluation Example

Let's break down the expression: 10 + 5 - 8 / 2

# Original Expression
result = 10 + 5 - 8 / 2
print("Result: {}".format(result))
                            
Result: 11.0

Detailed Steps:

  1. 8 / 2 (Division has higher precedence than Addition/Subtraction)
    Result: 4.0
  2. 10 + 5 (Addition and Subtraction have same precedence, Left-to-Right associativity)
    Result: 15
  3. 15 - 4.0
    Result: 11.0

6. Comparison Operators: The Decision Makers

Also known as Relational Operators, these symbols compare two values and return a Boolean result (True or False).

Operator Description Example Result
== Equal to 5 == 5 True
!= Not equal to 5 != 10 True
> Greater than 10 > 5 True
< Less than 3 < 7 True
>= Greater than or equal to 8 >= 8 True
<= Less than or equal to 4 <= 9 True 
a = 15
b = 10
print("a > b: {}".format(a > b))
print("a == b: {}".format(a == b))
print("a != b: {}".format(a != b))
                            
a > b: True
a == b: False
a != b: True

7. Logical Operators: Combining Conditions

These operators are used to combine conditional expressions (which evaluate to True or False) and return a single Boolean result. They are essential for complex decision-making in programs.

Operator Description Example Result
and True if BOTH are True (5 > 3) and (10 < 20) True
or True if ONE is True (5 > 10) or (10 < 20) True
not Reverses the result not (5 > 3) False 
is_sunny = True
is_warm = False

print("is_sunny and is_warm: {}".format(is_sunny and is_warm))
print("is_sunny or is_warm: {}".format(is_sunny or is_warm))
print("not is_sunny: {}".format(not is_sunny))
                            
is_sunny and is_warm: False
is_sunny or is_warm: True
not is_sunny: False

8. Assignment Operators: Efficient Value Updates

These operators are used to assign values to variables. The basic assignment operator is =, but Python offers shorthand versions that combine an arithmetic operation with assignment for more concise code.

Shorthand Assignment Operators

Operator Example Equivalent to Description
= x = 5 x = 5 Assigns value to variable
+= x += 3 x = x + 3 Add and assign
-= x -= 2 x = x - 2 Subtract and assign
*= x *= 4 x = x * 4 Multiply and assign
/= x /= 2 x = x / 2 Divide and assign (results in float)
%= x %= 3 x = x % 3 Modulus and assign
**= x **= 2 x = x ** 2 Exponentiate and assign
//= x //= 2 x = x // 2 Floor divide and assign 
counter = 10
counter += 5 # counter becomes 15
print("Counter: {}".format(counter))

price = 20.0
price *= 0.8 # price becomes 16.0
print("Price: {}".format(price))

data = 17
data %= 5 # data = 17 % 5 = 2 (remainder)
print("Data: {}".format(data))
                            
Counter: 15
Price: 16.0
Data: 2

9. Identity Operators

Compare if two variables refer to the exact same object in memory, not just if their values are equal.

Operator Description
is True if both are the same object
is not True if both are not the same object 
list1 = [1, 2, 3]
list2 = [1, 2, 3]
list3 = list1

print("list1 == list2: {}".format(list1 == list2)) # Checks value equality
print("list1 is list2: {}".format(list1 is list2))   # Checks object identity
print("list1 is list3: {}".format(list1 is list3))
                            
list1 == list2: True
list1 is list2: False
list1 is list3: True

10. Membership Operators

Test if a specific value is present within a sequence (like a string, list, or tuple).

Operator Description
in True if value is in sequence
not in True if value is not in sequence 
fruits = ["apple", "banana", "cherry"]
print("'apple' in fruits: {}".format("apple" in fruits))
print("'grape' in fruits: {}".format("grape" in fruits))

message = "Hello Python"
print("'Py' in message: {}".format("Py" in message))
                            
'apple' in fruits: True
'grape' in fruits: False
'Py' in message: True

11. Bitwise Operators (Brief Mention)

These operators work on the individual bits of binary numbers. They are powerful but used in more specialized areas like system programming, graphics, or data compression, and are less common for everyday application development.

Operator Name Description
& AND Sets bit to 1 if both bits are 1
| OR Sets bit to 1 if one of two bits is 1
^ XOR Sets bit to 1 if only one of two bits is 1
~ NOT Inverts all the bits
<< Zero fill left shift Shifts left, pushing zeros from right
>> Signed right shift Shifts right, pushing copies of leftmost bit from left
