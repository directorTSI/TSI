1. How a Computer Program Work

A program makes a computer usable. Without a program, a computer, even the most powerful, is nothing but an object.

Imagine that you want to know the average speed you've reached during a long journey. You know the distance, you know the time, you need the speed.

Naturally, the computer will be able to compute this, but the computer is not aware of such things as distance, speed, or time. Therefore, it is necessary to instruct the computer to:

  1. accept a number representing the distance;
  2. accept a number representing the travel time;
  3. divide the former value by the latter and store the result in the memory;
  4. display the result (representing the average speed) in a readable format.
  5. Repeat/loop the process

These five simple actions form a program. Of course, these examples are very far from what the computer can understand, but they are good enough to be translated into a language the computer can accept.

Language is the keyword.

2. Natural vs. Programming Languages

Language can be defined as a 'way of expressing thoughts.' There are many languages all around us. Some of them require neither speaking nor writing, such as body language; it's possible to express your deepest feelings very precisely without saying a word.

Another language you use each day is your mother tongue, which you use to manifest your will. Computers also have their own language, called machine language, which is limited to various principles.

Natural Languages

Examples: English, Swahili, Spanish. Evolve naturally. Used for human communication.

  • Flexible and often ambiguous
  • Rich vocabulary and expressions
  • Context-dependent meaning

Programming Languages

Examples: Python, Java, C++. Human-developed languages designed to give precise instructions to computers. Simpler grammar than natural languages.

  • Strict syntax rules
  • Unambiguous meaning
  • Designed for logical operations

It is important to note that machine languages are developed by humans. No computer is currently capable of creating a new language. However, that may change soon. Just as people use several very different languages, machines have many different languages too.

The difference, though, is that human languages are developed naturally. Moreover, they are still evolving, and new words are created every day as old words disappear. These languages are called natural languages.

3. Language Building Blocks

Every language, whether human or computer, has fundamental components:

Alphabet

A set of symbols used to build words (e.g., the Latin alphabet for English).

Lexis (Dictionary)

A set of words the language offers its users (e.g., "computer" from the English dictionary).

Syntax

A set of rules used to determine if a certain string of words forms a valid sentence (e.g., "I am a python" is correct).

Semantics

A set of rules determining if a phrase makes sense (e.g., "I ate a doughnut" makes sense, but "A doughnut ate me" doesn't).

4. High-Level Languages

We need a language in which humans can write their programs and a language that computers may use to execute the programs, one that is far more complex than machine language and yet far simpler than natural language.

Such languages are often called high-level programming languages. They are at least somewhat similar to natural ones in that they use symbols and words readable to humans.

A program written in a high-level programming language is called a source code (in contrast to the machine code executed by computers). Similarly, the file containing the source code is called the source file.

5. Compilation vs. Interpretation

Computer programming is the process of writing code in a chosen language to make a computer do what you want. The outcome depends on the programmer's creativity, knowledge, and experience.

Of course, such a composition has to be correct in many senses:

  • alphabetically - a program needs to be written in a recognizable script, such as Roman, Cyrillic, etc.
  • lexically - each programming language has its dictionary, and you need to master it; thankfully, it's much simpler and smaller than the dictionary of any natural language;
  • syntactically - each language has its rules and they must be obeyed;
  • semantically - the program has to make sense.

Unfortunately, a programmer can also make mistakes with each of the above four senses. Each of them can cause the program to become completely useless.

Let's assume that you've successfully written a program. How do we persuade the computer to execute it? You have to render your program into machine language. Luckily, the translation can be done by a computer itself, making the whole process fast and efficient.

Compilation

The source code is translated into a machine-readable file (executable) before execution. This is done by a compiler.

  • Entire program translated at once.
  • If code changes, it must be recompiled.
  • Faster execution once compiled.

Interpretation

The program performing this kind of transformation is called an interpreter, as it interprets the code every time it is intended to be executed. It also means that you cannot just distribute the source code as-is, because the end-user also needs the interpreter to execute it.

  • Translates and executes line by line.
  • No separate compilation step needed to run.
  • Easier to debug, but can be slower.
  • Python is an interpreted language!

Due to some very fundamental reasons, a particular high-level programming language is designed to fall into one of these two categories. There are very few languages that can be both compiled and interpreted.

7. Introducing Python

Python is a widely used, interpreted, object-oriented, and high-level programming language with dynamic semantics, used for general-purpose programming.

Fun Fact: Python was named after the BBC comedy series "Monty Python's Flying Circus," not the snake!

The Creator: Guido van Rossum

Born in 1956 in Haarlem, Netherlands. Guido van Rossum is the original author of Python.

While thousands contribute to Python's development, the initial vision was Guido's. He served as Python's "Benevolent Dictator For Life" (BDFL) until 2018, overseeing decisions about the language's evolution.

8. Python's Core Philosophy

In 1999, Guido van Rossum defined his goals for Python:

Easy & Intuitive

Just as powerful as major competitors, but simple to pick up. Anyone can learn to use it effectively.

Open Source

Anyone can use it and contribute to its development. Completely free for all purposes.

Readable Code

Code should be as clear as plain English, emphasizing readability and maintainability.

Rapid Development

Suitable for everyday tasks, allowing for short development times and quick prototyping.

9. Why Choose Python?

There are many reasons we've listed some of them already, but let's enumerate them again in a more practical manner:

Easy to Learn

The time needed to learn Python is shorter than for many other languages; this means that it's possible to start the actual programming faster.

Easy to Understand

It's also often easier to understand someone else's code faster if it is written in Python.

Easy to Obtain, Install, and Deploy

Python is free, open, and multiplatform (works on Windows, macOS, Linux, etc.).

Global Popularity

Python is the most popular programming language in the world, or at least ranks very high in the top three. Its versatility and ease of use make it a top choice for various applications.
