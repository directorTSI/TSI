Introduction: Conditional Statements

Conditional statements are essential programming constructs that allow your Python programs to make decisions and execute different blocks of code based on whether certain conditions are True or False. They introduce logic and adaptability into your applications.

This lesson will cover the various forms of conditional statements in Python, including `if`, `if-else`, `if-elif-else`, and nested `if` statements, along with best practices for writing clean and effective conditional logic.

1. The `if` Statement: Simple Decisions

The if statement is the most basic conditional. It executes a block of code ONLY if a specified condition evaluates to True. If the condition is False, the if block is skipped.

Syntax

# Syntax for if statement
# if condition:
#     # Code to execute if condition is True
#     statement_1
#     statement_2
# # Code outside the if block (always executes after if)

# Remember the colon : after the condition and the indentation for the code block!

Example 1: Check if Positive

If a number is positive, print a message.

num = 10
if num > 0:

num = -5

2. The `if-else` Statement: Two Paths

The if-else statement provides two possible execution paths. If the if condition is True, the if block executes. Otherwise (if False), the else block executes.

Syntax

# Syntax for if-else statement
# if condition:
#     # Code if condition is True
# else:
#     # Code if condition is False

# The 'else' block provides a default action when the 'if' condition isn't met.

3. The `if-elif-else` Statement: Multiple Choices

When you have more than two possible conditions to check, the `if-elif-else` chain is used. Python checks conditions sequentially. The first `True` condition's block is executed, and the rest of the chain is skipped. The `else` block acts as a fallback if no `if` or `elif` conditions are met.

Syntax

# Syntax for if-elif-else statement
# if condition_1:
#     # Code if condition_1 is True
# elif condition_2:
#     # Code if condition_1 is False AND condition_2 is True
# elif condition_3:
#     # Code if condition_1, condition_2 are False AND condition_3 is True
# else:
#     # Code if all above conditions are False

# You can have as many 'elif' (short for 'else if') blocks as needed.

4. Nested `if` Statements: Conditions Within Conditions

A nested if statement is an if statement that is placed inside another if, elif, or else block. This allows for more complex, hierarchical decision-making, where one condition must be true before a secondary condition is evaluated.

Syntax

# Syntax for Nested if statements
# if outer_condition:
#     # Code for outer condition is True
#     if inner_condition_1:
#         # Code if outer is True AND inner_1 is True
#     else:
#         # Code if outer is True AND inner_1 is False
# else:
#     # Code if outer condition is False

# Indentation is critical to define the scope of each nested block.

Example 1: Discount Eligibility

Apply a discount if a customer is a VIP AND makes a large purchase.

is_vip = True
purchase_amount = 120

if is_vip:
    print("Eligible for VIP benefits.")
    if purchase_amount > 100:
        print("Applying extra discount for large purchase!")
    else:
        print("Standard VIP discount.")
else:
    print("No VIP benefits.")

Applying extra discount for large purchase!

5. Best Practices for Conditionals

Writing clear, efficient, and maintainable conditional statements is key to robust code.

  • Proper Indentation: Always use consistent indentation (4 spaces is standard) to define code blocks. Incorrect indentation leads to IndentationError.
  • Clear Conditions: Make your conditions (>, ==, and, or) as simple and readable as possible. Avoid overly complex single conditions.
  • Use Logical Operators: For combining multiple checks, and and or can often simplify nested if's (e.g., if age > 18 and has_license:).
  • Order Matters: In if-elif-else chains, the order of elif conditions is crucial as Python executes the first True block it encounters.
  • Readability: If conditions become very long or complex, consider breaking them into smaller, more readable variables.
  • The else Fallback: Always consider an else block to handle cases where none of your if or elif conditions are met, preventing unexpected behavior.
